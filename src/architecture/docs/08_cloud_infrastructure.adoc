=== Network Security Group (NSG) Rules

TIP: Already aligned to your HLD approach, where NSGs control *spoke traffic*

*Details to capture:*

* Source → Destination
* Port / Protocol
* Action (Allow/Deny)
* Priority
* Purpose


.NSG Rules
====
*Example table:*

[width="88%",cols="28%,16%,20%,8%,11%,17%",options="header",]
|===
|*RULE NAME* |*SOURCE* |*DESTINATION* |*PORT* |*ACTION* |*PURPOSE*
|Allow-App-To-DB |App Subnet |DB Subnet |1433 |Allow |SQL access
|Allow-App-To-Blob |App Subnet |Storage Subnet |443 |Allow |Blob Access
|===
====


IMPORTANT: If traffic stays *inside the VNet* → *NSG rules must be defined*

IMPORTANT: If traffic leaves the VNet → *NSGs are not the primary control* (handled via hub firewall(s))

[width="92%",cols="39%,21%,40%",options="header",]
|===
|*TRAFFIC TYPE* |*NSG REQUIRED* |*NOTE*
|Subnet → Subnet (same VNet) |✅ Yes |Primary control
|VNet → External / Internet |❌ No |Controlled by firewall/routing
|VNet → Other VNet |❌ No |Controlled via peering {plus} firewall
|===
